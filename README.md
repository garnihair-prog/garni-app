# GARNI アプリ

美容室 GARNI 向けの業務アプリです。お客様向け予約・マイページと、スタッフ向け管理画面（予約管理・顧客カルテ・シフト管理）を、実際に動くデータベース付きで提供します。

## 特徴

- **追加インストール不要**：Python 3 の標準ライブラリのみで動作します（Flask/Djangoなどのフレームワークも、npm/Node.jsも不要）。`python3 server.py` を実行するだけで起動します。
- **実データベース**：SQLite（`garni.db`）にすべてのデータが保存されます。予約・顧客・カルテ・シフトのいずれも、ページを再読み込みしても消えません。
- **二重予約防止**：メニューの所要時間（カット60分・カラー120分・パーマ120分・トリートメント30分・縮毛矯正180分）をもとに、スタイリストの予約時間が重複しないようにチェックします。営業時間・シフトの休み・定休日も考慮されます。
- **営業時間・定休日を設定画面から変更可能**：スタッフ画面の「設定」で開店/閉店時刻と定休日（曜日単位）を管理でき、お客様向けの予約カレンダー・空き状況チェックにすぐ反映されます。
- **不定休（臨時休業日）にも対応**：曜日が決まっていない休みは、スタッフ画面の「設定」→「臨時休業日」でカレンダーから日付を指定して登録できます。登録した日はお客様の予約カレンダーで自動的に選択できなくなります。
- **メニューごとの最終受付時間を設定可能**：閉店時刻より早く受付を締め切りたいメニューは、スタッフ画面の「設定」→「メニューごとの最終受付時間」で設定できます（初期設定：カット17:00、カラー・パーマ16:00、縮毛矯正15:00）。パーマとカラーを同時に予約する場合専用の最終受付時間（初期設定15:00）も別途設定でき、選択したメニューに応じてお客様向けの予約時間選択にすぐ反映されます。
- **お客様分析（メニューカテゴリー別・男女別・年齢別）**：予約時にお客様が任意で入力した性別・年齢と、実際の予約データをもとに、スタッフ画面のダッシュボードでメニューカテゴリーごとのお客様数・割合、男女別・年齢層別の構成を確認できます。
- **メニュー・価格をスタッフ画面から編集可能**：スタッフ画面の「メニュー・価格」で、メニューの追加・名前/説明/価格/所要時間の変更・削除ができます。変更内容はお客様向けの予約画面にすぐ反映されます。髪の長さなどで料金が変わるメニューは「目安価格（〜表示）」にチェックすると、お客様向け画面で「¥6,600〜」のように表示できます。同意書が必要なメニューには「同意書」欄で対象の同意書を割り当てられます（詳細は下記）。
- **学割の割引額をメニューごとに設定可能**：スタッフ画面の「メニュー・価格」で「学割引額」に金額を入れると、お客様向けの価格の下に「学割 -¥500」のように表示されます（初期設定ではカラー・パーマが各500円引き）。金額は表示のみで、実際の割引は当日学生証をご提示いただいた際に会計で適用してください。
- **なりたいスタイルの写真＋施術後の写真でスタイル記録を管理**：お客様は予約時に「なりたいスタイル」の参考写真を任意でアップロードでき、来店後はスタッフが顧客カルテに施術後の写真を追加できます。希望スタイルと仕上がりが並べて記録されるので、次回来店時の参考になります。
- **マイページから予約の追記・写真追加が可能**：お客様はマイページ（電話番号で照会）から、来店予定（予約済み）の内容に「ご要望・リクエスト」の追記や「希望スタイル写真」の追加ができます。来店履歴では、希望スタイル写真と施術後写真を並べて確認できます。
- **マイページから予約のキャンセルが可能**：お客様はマイページから、来店予定の予約を自分でキャンセルできます（来店直前でも可能）。土日祝のご予約は、設定しているキャンセルポリシーに応じてキャンセル料が自動計算され、予約管理・顧客カルテに記録されます。お客様がキャンセルすると、設定していればオーナー様へ通知メールが届きます（新規予約時の通知と同じ仕組み）。
- **マイページで紹介クーポンの受け取り・使用状況を確認可能**：お客様はマイページ内に、自分が獲得した紹介クーポンの一覧（誰を紹介して発行されたか、金額、利用可能／使用済み／期限切れのステータス）を確認できます。従来の「紹介」タブと同じ情報を、予約状況の確認と同じ画面でまとめて見られます。
- **予約完了時にカレンダーへ追加してリマインダー登録**：予約完了画面・マイページの来店予定から「カレンダーに追加」ボタンで、お使いのスマホのカレンダーアプリ（iPhone／Googleカレンダー等）に予約日時を登録できます。前日に通知が届くよう設定済みです（実際の通知はお使いのカレンダーアプリの設定に従います）。
- **新規予約が入るとオーナー様へメール通知**：予約が入るたびに、指定したメールアドレスへ自動で通知メールを送信できます（任意設定。詳細は下記「新規予約のメール通知を設定する」を参照）。設定しない場合も予約自体には影響ありません。
- **キャンセル料を自動計算**：スタッフ画面の「設定」→「キャンセルポリシー」で設定した割合に基づき、キャンセル料を自動計算して記録します。
  - お客様都合の事前キャンセル（マイページからのキャンセル、または予約管理で「キャンセル」にした場合）：予約日が土曜・日曜・日本の祝日の予約のみ対象です。前日のキャンセルは前日用の割合（初期値50%）、当日のキャンセルは当日用の割合（初期値100%）、2日以上前のキャンセルは無料です。平日の予約は対象外（無料）です。
  - 無断キャンセル（予約管理で「無断キャンセル」にした場合）：平日・土日祝を問わず、毎回当日用の割合（初期値100%）を適用します。
  - 会計は手動でご案内ください（自動請求は行いません）。
- **当日予約は1時間先まで受付**：当日のご予約は、現在時刻から1時間後以降の時間帯のみお選びいただけるようになっています（直前すぎる予約の防止）。
- **予約確認画面に免責事項を表示**：お客様が予約を確定する直前の確認画面に、施術結果・キャンセルポリシー（設定したキャンセル料の割合が自動で反映されます）・その他注意事項（体質・天災等によるやむを得ない変更・お写真のSNS等使用について）の免責事項を表示します。内容に同意するチェックを入れないと予約を確定できない仕組みです。
- **カラー・ブリーチ／パーマ・縮毛矯正の同意書にクリックで同意**：スタッフ画面の「メニュー・価格」で、メニューごとに同意書（初期設定：「ブリーチ・カラー施術同意書」「パーマ・縮毛矯正施術同意書」）を割り当てられます。お客様が予約時に対象メニューを選ぶと、免責事項の確認画面の前に同意書専用の画面が表示され、本文を確認して「同意します」にチェックを入れないと次に進めません。パーマ＋カラーなど複数の同意書が必要な組み合わせの場合は、1枚ずつ順番に同意していただきます。誰が・いつ・どの同意書に同意したかは予約に記録され、スタッフ画面の顧客カルテ（お客様詳細）から確認できます。同意書の文面は今後追加・変更が必要な場合はご相談ください。
- **ホーム画面にアイコン追加OK**：Webアプリマニフェスト・アイコン画像を用意済みなので、Apple App Store／Google Playへの登録なしで、スマホのホーム画面にアプリのように追加できます（詳細は下記「ホーム画面にアイコンを追加する」を参照）。

## 起動方法（ローカルで試す）

```bash
cd garni-app
python3 server.py 8000
```

起動したら、ブラウザで以下にアクセスします。

- お客様向け： http://localhost:8000/
- スタッフ管理画面： http://localhost:8000/staff/ （初期パスワード：`garni2026`）

初回起動時に `garni.db` が自動生成され、サンプルデータ（メニュー・スタイリスト・顧客・予約・シフト）が投入されます。データをまっさらな状態からやり直したい場合は `garni.db` を削除してから再起動してください。

### スタッフパスワードの変更

環境変数 `GARNI_STAFF_PASSWORD` で変更できます。

```bash
GARNI_STAFF_PASSWORD="好きなパスワード" python3 server.py 8000
```

### 新規予約のメール通知を設定する

新しい予約が入るたびに、指定したメールアドレスへ自動で通知メールを送信できます。以下の環境変数を設定してください（Gmailを利用する場合の例です）。未設定の場合はメール通知機能はスキップされ、予約自体には影響しません。

```bash
GARNI_SMTP_USER="送信元にするGmailアドレス@gmail.com"
GARNI_SMTP_PASSWORD="Gmailのアプリパスワード（16桁）"
GARNI_NOTIFY_EMAIL="通知を受け取りたいメールアドレス（未設定ならGARNI_SMTP_USER宛に送信）"
```

Gmailの「アプリパスワード」は、Googleアカウントの設定（セキュリティ→2段階認証を有効にした上で「アプリパスワード」）から発行できます（通常のGmailログインパスワードとは別物です）。Render.comにデプロイする場合は、ダッシュボードの「Environment」タブでこれらの環境変数を追加してください。

## ホーム画面にアイコンを追加する（アプリのように使う）

公開後のURLをスマホのブラウザで開くと、ネイティブアプリのようにホーム画面にアイコンを追加できます（App Store等への登録は不要です）。

### iPhone（Safari）

1. お客様向けページ（例：`https://（公開したURL）/`）をSafariで開く
2. 下部の共有アイコン（□に↑）をタップ
3. 「ホーム画面に追加」を選択
4. 名前を確認して「追加」をタップ

→ ホーム画面にGARNIのアイコン（茶色地に白のG）が追加され、タップするとアドレスバーなしのアプリのような画面で開きます。

### Android（Chrome）

1. お客様向けページをChromeで開く
2. 右上の「⋮」メニューを開く
3. 「アプリをインストール」または「ホーム画面に追加」を選択

スタッフ管理画面（`/staff`）も同様の手順でホーム画面に追加できます（アイコンは「GARNIスタッフ」という名前で追加されます）。

## QRコードでお客様にご案内する

お客様向け予約ページ（公開URLの「/」）へのQRコードカードを用意しています（`tools/` フォルダ参照）。店頭・レジ・ショップカードなどに掲示すれば、お客様がスマホのカメラで読み取るだけで予約画面を開けます。デプロイして公開URLが決まったら、Claudeに「QRコードカードを作って」と依頼してください（URLを伝えるだけで、ブランドカラー入りのQRコード画像をその場で作成します）。

## 無料〜低コストで公開する方法（推奨）

このアプリはPythonの標準ライブラリだけで動くので、Python 3が動く環境ならどこでも公開できます。代表的な選択肢です。

1. **Render.com（無料プランあり・要注意）**：GitHubにこのフォルダをpushし、"Web Service" として登録。起動コマンドを `python3 server.py $PORT` に設定するだけで公開できます。**ただし無料プランは「15分アクセスがないと自動でスリープし、次回アクセス時にはローカルディスクの内容（`garni.db` を含む）が全て消えて初期状態に戻る」仕様です**。つまり、営業時間・メニュー・予約データなど、スタッフ画面で保存した内容がしばらく経つと消えてしまいます。有料のStarterプラン（月$7〜）＋永続ディスク（$0.25/GB/月）に切り替えるとこの問題は解消します。
2. **自宅サロンPCまたはNASで常時起動（推奨・データが消えない）**：Python 3さえ入っていれば、サロンのPCで `python3 server.py 8000` を実行しっぱなしにするだけでも運用できます。Synology／QNAP等のDockerが使えるNASをお持ちの場合は、本アプリに同梱の `Dockerfile` と `docker-compose.yml` を使って常時起動できます（下記「NAS（Docker）で常時起動する」を参照）。自宅の回線を通じて外出先からも予約を受け付けたい場合は、Cloudflare Tunnel等の利用も検討してください。
3. **PythonAnywhere（無料プランあり）**：小規模な個人利用に向いています。ファイルをアップロードし、WSGI経由で動かす場合は多少の設定調整が必要です。こちらも無料プランのディスク永続性は事前にご確認ください。

将来的にアクセスが増えてSQLiteでは力不足になった場合は、`db.py` の接続部分をPostgreSQL（例：Supabaseの無料枠）に差し替えることでスケールできます。テーブル定義（`SCHEMA`変数）はほぼそのまま移植可能です。

### データの保存先を変更する（`GARNI_DATA_DIR`）

環境変数 `GARNI_DATA_DIR` を設定すると、データベース（`garni.db`）とアップロード写真の保存先を、アプリ本体とは別のフォルダに変更できます。Dockerなどで永続ボリュームをマウントする場合に使用します（未設定の場合は今まで通りアプリと同じフォルダに保存されます）。

```bash
GARNI_DATA_DIR="/path/to/persistent/folder" python3 server.py 8000
```

### NAS（Docker）で常時起動する

Synology・QNAPなど、Dockerが使えるNASであれば、同梱の `Dockerfile` と `docker-compose.yml` を使って常時起動できます。`docker-compose.yml` は `./data` フォルダ（NAS上）に `garni.db` とアップロード写真を保存する設定になっているため、コンテナを再起動・再ビルドしてもデータは消えません。

Synology Container Managerでの手順の概要：

1. `garni-app` フォルダ一式（`Dockerfile`・`docker-compose.yml`を含む）を、共有フォルダ（例：`docker/garni-app`）にアップロードする。
2. Container Manager →「プロジェクト」→「作成」で、アップロードしたフォルダを指定する（`docker-compose.yml` が自動検出されます）。
3. 「次へ」を進めるとビルドが始まり、完了すると自動的に起動します。
4. NAS内から `http://NASのIPアドレス:8000/` にアクセスして動作確認してください。

外出先（自宅Wi-Fi以外）からもお客様が予約できるようにする場合は、ルーターのポート開放よりも「Cloudflare Tunnel」（無料）の利用をおすすめします。自宅のIPアドレスを公開せずに安全に公開でき、独自ドメイン（年間数百円〜）と組み合わせて使います。具体的な設定はお持ちのドメインの有無によって変わるため、必要であればClaudeに相談してください。

**写真データについての注意**：アップロードされた写真は `garni.db`（データベース）と同様に、`GARNI_DATA_DIR`（未設定の場合は `static/uploads/`）フォルダにファイルとして保存されます。

## ディレクトリ構成

```
garni-app/
├── server.py          # サーバー本体（ルーティング・API・認証）
├── db.py               # データベーススキーマ・初期データ
├── jp_holidays.py       # 日本の祝日を計算するモジュール（キャンセル料判定に使用）
├── Dockerfile           # Docker用のイメージ定義（NAS等での常時起動に使用）
├── docker-compose.yml   # Docker Compose設定（データ永続化・環境変数を設定）
├── garni.db             # SQLiteデータベース（初回起動時に自動生成）
└── static/
    ├── shared/styles.css        # 共通デザイン
    ├── customer/                 # お客様向けアプリ（予約・マイページ）
    ├── staff/                    # スタッフ管理画面（要ログイン）
    ├── icons/                    # ホーム画面追加用アイコン（各サイズ）
    ├── manifest.json             # お客様向けアプリのWebアプリマニフェスト
    ├── staff-manifest.json       # スタッフ管理画面のWebアプリマニフェスト
    ├── favicon.ico
    └── uploads/                   # アップロードされた写真（希望スタイル写真・施術後写真、自動生成）
```

## 主な機能

### お客様向け（ログイン不要）

- メニュー一覧・予約フロー（日付→スタイリスト→時間→お客様情報→確認）※当日のご予約は現在時刻から1時間後以降の時間帯のみ選択可能
- 予約時に「なりたいスタイルの写真」を任意でアップロード可能
- 予約完了画面から「カレンダーに追加」でスマホのカレンダーアプリにリマインダー登録
- 電話番号によるマイページ照会（予約履歴の確認、来店予定への追記・写真追加、来店履歴の希望スタイル写真と施術後写真の並び表示）
- お客様紹介：電話番号を照会すると自分専用の紹介コード・QRコードが表示され、お友達にシェアできます。そのQR/コードから来店したお友達のご来店が完了すると、紹介した方に500円割引クーポン（発行から6ヶ月間有効）を自動でプレゼントします。紹介できる人数に上限はありません。クーポンは1枚につき1回・1回のご来店（会計）につき1枚までしか使用できません。

### スタッフ向け（共有パスワードでログイン）

- ダッシュボード：本日の予約件数・売上実績・週間売上グラフ（実データ集計）、メニューカテゴリー別のお客様数・割合、男女別・年齢別のお客様構成
- 予約管理：日付ごとの予約一覧、お客様の希望スタイル写真のサムネイル表示、ステータス変更（未対応／来店済み／キャンセル／無断キャンセル、土日祝のキャンセル・無断キャンセル時は自動でキャンセル料を表示）。紹介クーポンを保有しているお客様の行には「紹介クーポンを適用」ボタンが表示され、会計時にその予約へ1枚だけ適用できます（同じ予約に2枚以上は適用できません）。
- 新規予約が入ると設定したメールアドレスへ自動通知（任意設定）
- 顧客カルテ：来店履歴・施術メモの自動蓄積（来店済みにすると自動でカルテに記録）、希望スタイル写真と施術後写真を並べて記録。あわせて、そのお客様の紹介コード・獲得した紹介クーポン（利用可能／使用済み／期限切れ）を表示し、会計時にクーポンを使用したら「使用済みにする」で記録できます。
  - 顧客カルテは、お客様が電話番号でご予約されると自動的に追加されます（手動での新規登録はできません）。
  - 転勤などで担当されなくなったお客様は、顧客カルテ画面の「お客様を削除」ボタンで顧客一覧から削除できます。過去の予約・カルテ・売上データは削除されず、ダッシュボードの集計にはそのまま残ります。同じ電話番号で再度ご予約が入った場合は、自動的に顧客一覧に復帰します。
- シフト管理：スタイリストごとの週間シフト編集
- メニュー・価格：メニューの追加・編集（名前／説明／価格／所要時間）・削除
- 設定：営業時間・定休日・臨時休業日・メニューごとの最終受付時間・キャンセルポリシー
